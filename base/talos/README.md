# Talos Machine Configuration

## What it does

This component contains the Talos Linux machine configuration patches that are applied on top of the base Talos machine config generated by `talosctl`. These patches customise the cluster nodes with:

- **`talosPatchConfig.yaml`** — common patches applied to all nodes (NTP, sysctls, kubelet extra args, etc.)
- **`talosPatchConfigControlplane.yaml`** — additional patches for control-plane nodes only
- **`systemVolumes.yaml`** / **`secondHddVolume.yaml`** — disk and volume configuration
- **`trustedRootsConfigTemplate.yaml`** — distributes the cluster CA to Talos nodes so they trust internal services

## Per-Node Patches

Individual nodes can have their talos machineconfig customised by placing a YAML patch file in the overlay:

```
overlays/<cluster>/talos/nodes/<hostname>.yaml
```

The hostname is the short name (first label of the FQDN, e.g. `brew-10` for `brew-10.example.com`).

These patches are deep-merged (patch wins) into the generated machine config **after** all role-level patches have been applied. Only the first YAML document (the main machine config) is merged — additional documents (HostnameConfig, etc.) are preserved unchanged.

Example — disable hugepages on a memory-constrained Raspberry Pi node:

```yaml
# overlays/tuborgnetes/talos/nodes/brew-10.yaml
machine:
  sysctls:
    vm.nr_hugepages: "0"
```

## Why it was added

Talos Linux requires machine configuration to be provided at boot time. This component holds the overlay-agnostic base patches. Cluster-specific machine configs are generated by `adminTasks/cluster-initialSetup.sh` and stored in `overlays/<cluster>/talos/`.

## Dependencies

None — this component contains static configuration templates that are consumed by `adminTasks/render-overlay.sh` and `talosctl` during cluster setup.

## Dependents

None — the generated Talos machine configs in `overlays/<cluster>/talos/` are consumed directly by `talosctl`, not by other Kubernetes components.

## User Guide

### Applying a configuration change to nodes

1. Edit the relevant patch file in `base/talos/`, in `overlays/<cluster>/talos/`, or in `overlays/<cluster>/talos/nodes/<hostname>.yaml` for a single node.
2. Re-render: `./adminTasks/render-overlay.sh overlays/<cluster>/<cluster>.env`
3. Apply the updated config to the nodes:
   ```bash
   talosctl apply-config --nodes <node-ip> --file rendered/<cluster>/talos/<node>.yaml
   ```

### Upgrading Talos

Update `TALOS_INSTALL_VERSION` in the overlay `.env` file, then:

```bash
talosctl upgrade --nodes <node-ip> --image factory.talos.dev/installer/<schematic-id>:<version>
```

### Upgrading Kubernetes

Update `KUBERNETES_VERSION` in the overlay `.env` file, then:

```bash
talosctl upgrade-k8s --to <version>
```
